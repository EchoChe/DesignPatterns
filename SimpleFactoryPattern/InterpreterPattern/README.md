解释器模型
解释器模型（Interpreter Pattern）提供了评估语言的语法或表达式方法，它属于行为型模式。这种模式实现了一个表达式接口，该接口解释一个特定的上下文。这种模式被用在SQL解析、符号处理引擎。

介绍：
	意图：给定一个语言，定义它的文法，并定义一个解释器，这个解释器使用该标识来解释语言中的句子。
	主要解决：对于一些固定文法构建一个解释句子的解释器。
	何时使用：如果一个特定类型的问题发生频率足够高，那么可能就值的将该问题的各个实例表述为一个简单语言中的句子。这样就可以构建一个解释器，该解释器通过解释这些句子来解决这些问题。
	如何解决：构建语法树，定义终结符和非终结符
	关键代码：构建环境类，包含解释器之外的一些全局变量，一般是HashMap
	应用实例：编译器、运算表达式计算
	优点：可扩展性好，灵活。 增加了新的解释器表达式的方式。 易于实现简单文法。
	缺点：可利用的场景比较少，对于复杂的文法比较难维护，解释器模式会引起类膨胀，解释器模式采用递归调用方法
	使用场景：1.可以将一个需要解释执行的预言中的句子表示为一个抽象语法树
			  2.一些重复出现的问题可以用一种简答的预言来进行表达
			  3.一个简单语法需要解释场景
     注意事项：可利用场景比较少
