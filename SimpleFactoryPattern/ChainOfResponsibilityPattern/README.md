责任链模式
	责任链模式是为了请求一种接收着对象的链。这种模式给予请求的类型，对请求的发送者和接受者进行解耦。这种类型的设计模式属于行为型模式。
	在这种模式中，通常每个接受者都包含对另一个接收者的引用。如果一个对象不能处理请求，那么他会把相同的请求传给下一个接收者，以此类推。

介绍
	意图：避免请求发送者与接收者耦合在一起，让多个对象都有可能接收请求，将这些对象连成一条线，并且沿着这条链传递请求，直到有对象处理为止。
	主要解决：责任链上的处理者负责请求，客户只需要将请求发送到职责链上即可，无需关心请求处理细节和请求的传递，所以职责连将请求的发送者和请求的处理者解耦了。
	何时使用：在处理消息时以过滤很多道。
	关键代码：Handler里面聚合它自己，在HanleRequest里面判断是否合适，如果没有达到条件则向下传递，向谁传递之前set进去。
	应用实例：击鼓传花
	优点：1.降低耦合度。它将请求的发送者和接收者解耦
		2.简化了对象。使得对象不需要知道链的结构
		3.增加给对象指派职责的灵活性，通过改变链内的成员或者调动他们的次序，允许动态的新增或者删除责任。
		4.增加新的请求处理类很方便。
	缺点：1.不能保证请求一定能被接受，系统性能受到一定影响，而且在进行代码调试时不太方便，可能会造成循环调用。可能不太容易观察运行时的特征，有碍于除去错。
	使用场景：1.由很多对象可以处理同一个请求，具体哪个对象处理该请求由运行时自动确定
			2.在不明确指定接收者的情况下，向多个对象中的一个提交一个请求
			3.可动态指定一组对象处理请求。
			
	实现：我们创建抽象类AbstarctLogger，带有详细的日志记录级别。然后我们创建三种类型的记录器，都扩展了AbstractLogger。每个记录器消息的级别是否属于自己的级别，如果是则相应的打印出来，
	否则将不打印并把消息传递给下一个记录器。
